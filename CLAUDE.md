# CardForge Online: 원시 카드 전략 서바이벌 통합 실행 명세

## 1. 프로젝트 개요
- **기술 스택**: Phaser 3 (Client), Node.js & Socket.io (Server)
- **장르**: 원시 카드 경영 & 전략 서바이벌 — 솔로 또는 1:1 PvP 선택 가능
- **저장**: 로컬 데이터(환경 설정, 점수 기록) 및 서버 세션(PvP 게임 상태)
- **인증**: 별도 가입 없는 임의 닉네임 기반 세션

---

## 2. 게임 모드

### 2.1 솔로 모드 (Solo)
- 닉네임 입력 후 즉시 단독 보드에서 시작.
- 서버 연결 없이 클라이언트 단독 실행.
- 약탈자(Raider)는 시간이 지남에 따라 자연 스폰됨 (PvP의 상대방 파견 없이 게임 내부 위협으로 등장).
- 스폰 주기: 게임 시작 후 60초마다 1마리, 이후 30초 간격으로 단축 (최소 20초).
- 패배 시 생존 시간과 조합 횟수(점수)를 결과 화면에 표시.

### 2.2 PvP 모드 (1:1 Online)
- 닉네임 입력 후 로비에서 방 생성/입장.
- Socket.io 기반 실시간 대전.
- 약탈자는 상대방이 `전사 + 배` 조합 성공 시 파견됨 (자연 스폰 없음).
- 게임 종료 시 양측 모두 조합 횟수(점수)를 결과 화면에 표시.

---

## 3. 매칭 및 방 시스템 (Lobby — PvP 전용)
- **Entry**: 유저가 닉네임을 입력하고 모드 선택 (솔로 / PvP).
- **Room (PvP)**:
  - 방 생성 시 제목 및 비밀번호(옵션) 설정.
  - 1:1 전용으로 2인 입장 시 게임 시작 가능.
- **Identity**: 클라이언트별로 `host`와 `guest` 역할을 부여받아 보드 위치 결정.

## 4. 화면 구성 및 UI 흐름

### 4.1 로비 화면
```
┌─────────────────────────────────┐
│         CardForge Online        │
│                                 │
│   닉네임: [____________]        │
│                                 │
│   [ Solo Play ]  [ Multi Play ] │
│   [ Setting  ]  [ Tutorial   ] │
└─────────────────────────────────┘
```
- 닉네임 입력 후 각 버튼으로 이동.
- Setting: 음량, 그래픽 등 환경설정.
- Tutorial: 게임 규칙 안내 화면.

### 4.2 멀티플레이 방 화면
```
┌─────────────────────────────────┐
│         Multi Play              │
│                                 │
│   [ 방 만들기 ]                 │
│   [ 방 참여하기 ]               │
│                                 │
│   [ ← 뒤로가기 ]               │
└─────────────────────────────────┘
```
- **방 만들기**: 서버가 고유 방코드를 생성해 화면에 표시. 상대방이 코드로 입장 가능.
- **방 참여하기**: 방코드 입력란에 코드를 입력하면 해당 방에 참가.

### 4.3 방 대기실 화면
```
┌─────────────────────────────────┐
│  방코드: AB12CD                 │
│                                 │
│  ● UserA (방장)  [Ready ✓]     │
│  ● UserB (게스트) [Ready ✓]    │
│                                 │
│  방장: [ 게임 시작 ]            │
│  게스트: 방장의 시작을 기다리는 중 │
└─────────────────────────────────┘
```
- 두 플레이어 모두 Ready 상태일 때 방장의 **게임 시작** 버튼이 활성화.
- 시작 버튼 클릭 시 **3초 카운트다운** 후 게임 화면으로 전환.

### 4.4 게임 화면 레이아웃
```
┌──────────────────────────────────────────┐
│ [상대방 Stat 구역]                        │
│  인구: 3/8 | 성벽: 있음 | 도시: 0 | 👑12 │
├──────────────────────────────────────────┤
│                                          │
│           [ 필드 구역 ]                  │
│      (카드들이 배치·드래그되는 공간)      │
│                                          │
├──────────────────────────────────────────┤
│ [내 Stat 구역]                           │
│  식량: 5 | 인구: 2/5 | 도시: 0 | 성벽: 3장│
│  농지: 1/1 | 👑 왕국점수: 8/30           │
├──────────────────────────────────────────┤
│ [기타 구역]  [ 게임 나가기 ] [ 왕국 건설 ]│
└──────────────────────────────────────────┘
```

#### 구역별 상세
| 구역 | 위치 | 표시 내용 |
|------|------|-----------|
| **상대방 Stat** | 상단 | 인구 합산, 성벽 유무, 도시 수, 왕국 점수 (5초 갱신) |
| **필드** | 중앙 | 내 카드 전체. 드래그&드롭·스택 조작 가능 |
| **내 Stat** | 하단 | 식량, 인구/제한, 도시 수, 성벽 수(= 성벽 체력), 농지/제한, 왕국 점수/30 |
| **기타** | 최하단 | 게임 나가기, 왕국 건설 버튼(30점 달성 시 활성화) |

- 솔로 모드에서는 상대방 Stat 구역 대신 **생존 시간** 타이머를 표시한다.
- 왕국 건설 버튼은 왕국 점수 30점 미만이면 비활성화(grayed out) 상태로 항상 표시한다.

## 5. 핵심 게임 메커니즘

### 5.1 화면 레이아웃 (반응형)
- **솔로**: 전체 화면이 단일 보드. 상단에 생존 시간·인구 수·조합 횟수(점수) 표시.
- **PvP**:
  - **My Board (Bottom)**: 내가 직접 조작하는 영역. 드래그 & 드롭 가능.
  - **Opponent Board (Top)**: 상대방의 상황을 실시간 관전하는 영역. (ReadOnly)
  - **Global Status**: 중앙에 남은 시간, 양측 인구 수, 이벤트 로그 표시.

### 5.2 게임 시작 조건
- 게임 시작 시 **사람 카드 1장 + 전사 카드 1장 + 농지 카드 1장**을 기초 자원으로 지급한다.
- 초기 보드에는 이 세 장만 존재하며, 이후 자원을 조합해 확장해야 한다.

### 5.3 인구 제한 시스템
- **인구(Population)**: 사람 + 전사 + 궁수 카드의 총 수량.
- **기본 인구 제한**: 5명.
- 인구가 제한에 도달하면 인구 카드를 새로 생성하는 조합이 차단된다.

| 카드 | 인구 제한 증가 |
|------|--------------|
| 목조 가옥 1개 | +1 |
| 마을 1개 | +3 |
| 도시 1개 | +6 |

- 인구 제한 = 5 + (목조 가옥 수 × 1) + (마을 수 × 3) + (도시 수 × 6).
- 건물 카드가 파괴되거나 소모되면 제한도 즉시 감소하며, 초과 인구가 생기면 경고 표시만 하고 강제 제거는 하지 않는다.
- HUD에 현재 인구 / 제한을 `3/8` 형태로 표시한다.

### 5.4 농지 제한 시스템
- **기본 농지 제한**: 1개.
- 농지 카드가 제한을 초과해 생성되는 조합(씨앗 + 농지 등)은 제한 도달 시 차단된다.

| 카드 | 농지 제한 증가 |
|------|--------------|
| 마을 1개 | +1 |
| 도시 1개 | +2 |

- 농지 제한 = 1 + (마을 수 × 1) + (도시 수 × 2).
- HUD에 현재 농지 수 / 제한을 표시한다.

### 5.5 승리 및 패배 조건
| 모드 | 승리 | 패배 |
|------|------|------|
| 솔로 | 없음 (생존 시간 최대화) | 인구(사람 + 전사 + 궁수)가 0명 |
| PvP  | 상대방 인구를 모두 제거 **또는 왕국 건설 버튼 선착** | 내 인구가 0명 |

- **PvP 제한 시간**: 시간 종료 시 인구가 많은 쪽이 승리.
- **왕국 건설 (Kingdom)**: 왕국 점수가 30점에 도달하면 HUD에 "왕국 건설" 버튼이 활성화됨. 먼저 버튼을 누른 플레이어가 상대방 인구 수와 관계없이 즉시 승리.

### 5.6 공격 유닛 스폰

| 유닛 | 솔로 | PvP |
|------|------|-----|
| **약탈자(Raider)** | 게임 루프가 자연 생성 (60초 → 30초 → 20초 간격) | `전사 + 배` 조합 성공 시 상대 보드에 즉시 생성 |
| **투석기(Catapult)** | (레시피 별도 정의) | (레시피 별도 정의) |

- 공격 유닛은 스폰 후 자동으로 우선순위에 따라 타겟을 향해 부유 이동(Floating Move)한다.
- **제거 소요 시간**: 약탈자·투석기 각각 제거에 **10초** 소요.

### 5.7 전투 교환비 및 우선순위

#### 전투 기본 원칙
- **전사만 선제공격 가능**. 궁수는 선제공격 불가 — 방어 반응만 한다.
- 전사를 드래그하여 약탈자/투석기 위에 놓으면 즉시 교전 시작 (10초 카운트).
- 전사 1명이 교전 중이면 해당 전사는 **다른 적에게 "없는 취급"**됨 — 적은 다음 우선순위 타겟을 즉시 노린다.

#### 약탈자(Raider) 교환비

| 약탈자 vs | 소모 비율 | 비고 |
|-----------|-----------|------|
| 전사 | 1:2 (약탈자 1 = 전사 2 제거) | 기습 보너스 |
| 성벽 | 1:2 (성벽 카드 2장 제거 후 약탈자 소멸) | 성벽 카드 수 = 성벽 체력 |
| 성벽 + 궁수 동시 존재 | 1:1 (성벽 카드 1장만 제거) | 궁수의 지원 효과 |
| 궁수 | 1:3 (궁수 3명 소모) | 성벽 있으면 궁수 무피해, 성벽 카드만 제거 |
| 사람 | 1:5 (사람 5명 소모) | |

**약탈자 공격 우선순위**: 전사 → 성벽 → 궁수 → 사람

전사가 없으면 부족한 만큼 다음 우선순위(성벽 체력 / 궁수 / 사람)로 피해를 넘긴다.

#### 투석기(Catapult) 교환비

| 투석기 vs | 소모 비율 |
|-----------|-----------|
| 전사 | 2:1 (전사 2명 소모) |
| 궁수 | 1:1 (궁수 1명 소모) |
| 성벽 | 1:3 (성벽 카드 3장 제거) |
| 사람 | 1:5 (사람 5명 소모) |

**투석기 공격 우선순위**: 궁수 → 성벽 → 전사 → 사람

#### 궁수(Archer) 방어

| 궁수 vs | 소모 비율 | 비고 |
|---------|-----------|------|
| 약탈자 | 3:1 (궁수 3명으로 약탈자 1 제거) | 성벽 존재 시 궁수 무피해, 성벽만 깎임 |
| 투석기 | 1:1 | |

궁수는 적이 보드에 진입하면 자동으로 반응하며, 유저가 별도로 드래그할 필요 없다.

### 5.8 척후병 (Scout)

#### 생성 및 파견
- **목재 + 사람** → 척후병 (사람 소모, 목재 소모).
- **척후병 + 배** → 상대방 보드에 척후병 카드 파견 (재료 카드 소모).

#### 정찰 기능
- 척후병은 상대 보드에서 **30초** 동안 유지된다.
- 유지 중 **5초마다** 아래 정보를 내 HUD에 갱신한다:

| 수집 정보 |
|-----------|
| 성벽 현재 체력 |
| 전사 수 |
| 궁수 수 |
| 사람 수 |
| 식량 수 |
| 도시 수 |
| 왕국 점수 |

- 30초 경과 후 척후병 카드는 자동 소멸하며 정보 갱신도 중단된다.

#### 척후병 교환비

| 척후병 vs | 소모 비율 | 비고 |
|-----------|-----------|------|
| 전사 | 1:0 | 전사가 척후병을 차단 불가 |
| 궁수 | 1:1 (궁수 1명 소모로 제거) | |
| 성벽 | 1:1 (성벽 카드 1장 제거) | |
| 사람 | 1:1 (사람 1명 소모) | |

**척후병 공격 우선순위**: 성벽 → 궁수 → 사람  
전사는 척후병을 막지 못하므로 우선순위에서 제외된다.

## 6. 카드 스택 시스템

### 6.1 스택 기본 규칙
- 같은 타입의 카드는 기본적으로 하나의 스택(뭉치)으로 관리된다.
- 스택은 카드 한 장처럼 보드에 존재하며, **왼쪽 상단에 뱃지 숫자**로 수량을 표시한다 (포커 카드 더미 스타일).
- 수량이 1이면 뱃지를 표시하지 않는다.
- 카드를 개별적으로 분리해 보드 다른 위치에 놓으면 해당 위치에 별도 스택으로 존재한다.

### 6.2 스택 조작 방법

| 조작 | 동작 |
|------|------|
| **바로 드래그** | 스택 전체를 이동 |
| **1초 길게 누른 후 드래그** | 스택에서 카드 1장만 분리되어 드래그. 마우스를 떼면 해당 위치에 드롭 |
| **숫자키 1~9 입력 후 드래그** | 스택에서 해당 숫자만큼 카드를 분리해 이동. 스택 수량보다 큰 숫자는 전체 이동과 동일 |

### 6.3 스택 조작 상세 규칙
- **길게 누르기**: 마우스(또는 터치) 다운 후 1초 경과 시 분리 모드 진입. 분리 중임을 나타내는 시각 피드백(테두리 강조 등)을 제공한다.
- **숫자키**: 마우스 커서가 스택 위에 올라간 상태에서 숫자키를 누르면, 그 수만큼 분리된 서브스택이 커서를 따라다니며 드래그 상태가 된다. 이후 드롭으로 위치를 확정한다.
- 분리된 카드가 같은 타입의 다른 스택 위에 드롭되면 자동으로 합쳐진다.
- 조합을 위한 드롭(다른 타입 카드 위)은 기존 조합 시스템과 동일하게 처리된다. 이 때 조합에 사용되는 수량은 분리된 카드 수량 기준이다.

### 6.4 카드 데이터 모델 (스택 포함)
```json
{
  "id": "stack-uuid",
  "type": "WARRIOR",
  "count": 5,
  "ratioX": 0.3,
  "ratioY": 0.6
}
```
- `count`: 해당 스택의 카드 수량. 항상 1 이상.
- 스택 분리 시 원본 스택의 `count`를 차감하고 새 스택 객체를 생성한다.
- `count`가 0이 되면 스택 객체를 즉시 제거한다.

## 7. 자연 스폰 시스템

### 7.1 스폰 카드 및 확률

| 카드 | 스폰 빈도 | 스폰 형태 |
|------|-----------|-----------|
| 바위 | 높음 | 개별 1장 |
| 나무 | 높음 | 개별 1장 |
| 늑대 | 낮음 | 1~3마리 뭉치 (랜덤) |
| 곰  | 매우 낮음 | 개별 1마리 |

### 7.2 바위 / 나무 채집
- **바위 + 사람** → 돌 2장 생성 (사람 소모 안됨, 바위만 소모) — 비소모 조합
- **나무 + 사람** → 목재 2장 생성 (사람 소모 안됨, 나무만 소모) — 비소모 조합

### 7.3 늑대 (Wolf)

#### 스폰 규칙
- 1~3마리 범위에서 랜덤으로 스폰되며, **최대 3마리가 하나의 뭉치 스택**으로 등장한다.
- 4마리 이상이 동시에 스폰될 경우, 초과분은 개별 스택으로 필드에 별도 존재한다.

#### 전사로 처치
- **늑대 + 전사** 조합 시 **식량 1장** 생성.
- 전사 생존 확률:
  - `투여 전사 수 < 늑대 수` 인 경우: 전사 1마리당 `1 - (투여 전사 수 / 늑대 수)` 확률로 제거.
  - `투여 전사 수 ≥ 늑대 수` 인 경우: 투여된 전사 전원 생존.

#### 자동 소멸 타이머 (20초)
스폰 후 **20초** 내에 처리하지 않으면 늑대가 마을을 습격하고 소멸한다.  
습격 결과는 보드 상태에 따라 아래와 같이 결정된다.

| 보드 상태 | 습격 결과 |
|-----------|-----------|
| 성벽 + 궁수 둘 다 존재 | 성벽 카드 1장 제거, 식량 +늑대 수만큼 드랍 |
| 궁수만 존재 (성벽 없음) | 궁수 : 늑대 = 1:1 소모, 식량 +늑대 수만큼 드랍 |
| 성벽만 존재 (궁수 없음) | 사람 -늑대 수만큼 제거, 식량 드랍 없음 |
| 성벽·궁수 모두 없음 | 사람 -늑대 수만큼 제거, 식량 드랍 없음 |

### 7.4 곰 (Bear)
- 항상 **1개체**로 스폰된다.
- 전투력·습격 로직은 **늑대 3마리 뭉치와 동일**하게 처리한다.
- 그 외 타이머(20초), 처치 보상(식량 1장), 자동 소멸 습격 규칙은 늑대와 공통이다.

### 7.5 스폰 구현 지침
- 바위·나무는 게임 루프에서 가장 자주 스폰되며, 보드가 넘치지 않도록 최대 동시 존재 수를 제한할 수 있다.
- 늑대·곰 스폰 시 20초 카운트다운 타이머를 해당 카드에 부착하고, 만료 시 즉시 습격 처리 후 카드를 제거한다.
- 늑대 뭉치의 수량(`count`)은 스택 시스템과 동일한 구조로 관리하되, 최대값 3을 초과하면 새 스택으로 분리 생성한다.

## 9. 점수 시스템

### 9.1 조합 점수 (결과 화면용)
- **조합 점수**: 조합 성공 1회 = 1점. 모든 조합에 동일하게 적용.
- 게임 중 실시간으로 상단 HUD에 현재 조합 횟수를 표시한다.

### 9.2 왕국 점수 (Kingdom Score — PvP 즉시 승리 조건)
왕국 점수는 현재 보드 상태를 기반으로 **실시간 재계산**되는 점수다.

| 조건 | 왕국 점수 |
|------|-----------|
| 도시 1개 | +1 |
| 성벽 카드 10장당 | +1 |
| 인구(사람 + 전사 + 궁수) 5명당 | +1 |

- 왕국 점수 합계가 **30점** 이상이 되면 HUD에 **"왕국 건설" 버튼**이 활성화된다.
- 30점 미만으로 다시 떨어지면 버튼이 비활성화된다.
- PvP에서 버튼을 먼저 누른 플레이어가 즉시 승리 (상대방 인구 무관).
- 솔로 모드에서는 왕국 점수 30점 달성 시 최고 기록으로 저장하고 계속 플레이 가능.
- HUD에 현재 왕국 점수를 실시간으로 표시한다 (예: `👑 24/30`).

### 9.3 결과 화면 (공통)
게임 종료(승리·패배·시간 종료) 시 결과 화면을 노출한다.

| 항목 | 솔로 | PvP |
|------|------|-----|
| 조합 횟수(점수) | O | O (양측 모두) |
| 생존 시간 | O | X |
| 승/패 여부 | X (기록만) | O |
| 재시작 / 로비 버튼 | O | O |

- 솔로는 이전 최고 점수(로컬 저장)와 비교해 신기록 여부를 표시한다.
- PvP는 양측의 조합 점수를 나란히 보여주며 승자를 강조 표시한다.

### 10.3 정보 공개 수준 (PvP)

PvP에서 상대방 정보는 두 가지 채널로 공개된다.

| 채널 | 주기 | 공개 정보 |
|------|------|-----------|
| **기본 동기화** | 5초마다 자동 | 인구 수(전사+궁수+사람 합산), 성벽 유무, 도시 수, 왕국 점수 |
| **척후병 정찰** | 5초마다 (척후병 유지 중) | 위 항목 + 전사/궁수/사람 개별 수, 성벽 정확한 체력, 식량 수 |

- 기본 동기화는 서버가 5초마다 두 플레이어 모두에게 브로드캐스트한다.
- 척후병이 없으면 상대방의 인구 세부 구성(전사/궁수/사람 분리 수치)과 성벽 체력, 식량은 알 수 없다.

### 10.4 통신 이벤트 정의 (PvP 전용)
- **SEND_COMBINATION**: 클라이언트가 조합 성공 시 서버에 전송. 서버가 `combineCount` 증가 후 브로드캐스트.
- **SPAWN_RAIDER**: 서버가 공격을 받은 클라이언트에게 약탈자 생성을 명령.
- **SYNC_CARD_POS**: 내 카드의 위치 변화를 상대방 화면에 동기화.
- **SYNC_STATUS**: 서버가 5초마다 각 플레이어에게 상대방의 기본 정보(인구 합산, 성벽 유무, 도시 수, 왕국 점수)를 전송.
- **SCOUT_REPORT**: 척후병 유지 중 5초마다 상세 정보를 척후병 파견자에게 전송.
- **GAME_OVER**: 특정 유저의 인구가 0이 될 때 서버가 양측 `combineCount` 포함 결과 브로드캐스트.

## 11. 구현 우선순위
1. **모드 선택 화면**: 닉네임 입력 → 솔로 / PvP 분기.
2. **솔로 기본 루프**: 단일 보드, 자연 스폰 약탈자, 패배 감지.
3. **카드 스택 시스템**: 스택 렌더링(수량 뱃지), 전체 이동, 1초 길게 눌러 1장 분리, 숫자키 N장 분리.
4. **점수 시스템**: 조합 횟수 카운트 및 결과 화면 (솔로 최고 점수 로컬 저장 포함).
5. **네트워크 레이어**: Socket.io를 이용한 닉네임 기반 방 생성/입장 (PvP).
6. **동기화 엔진**: 드래그 이동(스택 포함)이 상대방 관전 화면에 실시간으로 반영.
7. **약탈자 파견**: PvP `전사 + 배` 조합 시 서버를 통해 상대방 보드에 약탈자 생성.
8. **전투 AI**: 약탈자 자동 부유 이동(Floating Move) 및 인구 파괴.
9. **결과 처리**: 승리/패배 조건 감시, 양측 조합 점수 비교 결과 UI.

## 12. AI 작업 가이드라인
- 인구를 생성하는 조합(사람 + 창 → 전사 등)은 실행 전 현재 인구 < 인구 제한 여부를 검사하고, 초과 시 조합을 거부하고 시각 피드백을 제공한다.
- 인구 제한은 보드 위 목조 가옥·마을·도시 카드 수량으로 실시간 재계산하며 별도 필드로 캐싱한다.
- 농지 제한도 마을·도시 카드 수량으로 실시간 재계산한다. 농지 생성 조합 전에 현재 농지 수 < 농지 제한 여부를 검사한다.
- 왕국 점수는 매 게임 루프 틱마다 재계산한다: `도시 수 × 1 + floor(성벽 카드 수 / 10) + floor(인구 수 / 5)`. 30점 이상이면 왕국 건설 버튼을 활성화, 미만이면 비활성화한다.
- PvP에서 왕국 건설 버튼 클릭은 서버가 수신한 시각 기준으로 선착 판정한다.
- 전투 교환비는 `CombatResolver` 모듈에서 테이블로 관리하며, 약탈자·투석기별 우선순위 목록과 피해량을 데이터로 분리한다.
- 교전 중인 전사는 `busy` 상태로 플래그를 세우고, 적 타겟 결정 시 `busy` 전사는 제외하고 다음 우선순위를 탐색한다.
- 성벽 카드 수가 곧 성벽 체력이다. 피해 발생 시 해당 수만큼 성벽 카드 스택에서 `count`를 차감하며, 0이 되면 스택을 제거한다. 성벽 + 궁수 동시 존재 여부는 피해 계산 전에 확인한다.
- 척후병은 상대 보드의 정보를 5초 주기로 수집해 소켓 이벤트(`SCOUT_REPORT`)로 자신에게 전송한다. 30초 타이머 만료 시 카드 제거 및 정보 갱신 중단. 전사의 `busy` 플래그와 무관하게 전사는 척후병을 차단할 수 없다.
- 늑대·곰 카드는 스폰 시 20초 타이머를 가지며, 만료 시 `습격 처리 → 카드 제거`를 게임 루프에서 일괄 처리한다. 타이머는 카드 데이터에 `expiresAt` 필드로 부착한다.
- 늑대와의 전사 교전 결과(전사 생존 확률)는 서버 또는 클라이언트에서 `Math.random()` 기반으로 판정하되, PvP에서는 서버가 최종 판정한다.
- 곰은 내부적으로 `strength: 3`으로 표현하고, 늑대 뭉치의 `count`를 그대로 전투 로직에 활용해 코드 중복을 피한다.
- `사람 + 식량` 조합은 결과 카드가 2장인 **다중 결과 조합**이다. 조합 테이블에 `result: [{ type: "PERSON", count: 2 }]` 형태로 정의하며, 인구 제한 검사는 조합 실행 전에 `현재 인구 + 1 ≤ 인구 제한` 여부로 판단한다.
- `농지 + 사람` 조합은 예외 처리가 필요한 **비소모 조합**이다. 사람 카드를 제거하지 않고 농지 카드의 `mode`를 `"producing"`으로 변경하는 방식으로 구현한다. 조합 테이블에 `consumeB: false` 플래그를 두어 일반 조합과 구분한다.
- 모든 조합 로직은 `CombinationEngine` 클래스에서 데이터 테이블 기반으로 처리한다.
- 솔로 조합 판정은 클라이언트에서, PvP 조합 판정은 서버에서 최종 검증(Validation)한다.
- 약탈자 스폰 로직은 모드(솔로/PvP)에 따라 분기하되 약탈자 자체의 AI 로직은 공통 모듈로 유지한다.
- 점수(`combineCount`)는 조합 성공 시점마다 즉시 갱신하며, 게임 종료 후에는 변경하지 않는다.
- 카드 스택의 `count`는 내부 로직(분리·합치기·조합 소모)에서만 변경하며 직접 UI 조작으로는 수정하지 않는다.
- 스택 분리 시 `count` 차감과 새 스택 생성은 원자적으로 처리해 수량 불일치가 발생하지 않도록 한다.
- 숫자키 분리는 현재 포커스(hover)된 스택에만 적용되며, 스택이 없거나 숫자가 0이면 무시한다.
- 반응형 UI를 위해 카드 위치값은 0~1 사이의 비율값(`ratioX`, `ratioY`)으로 전송한다.
- 새 기능을 추가할 때는 먼저 `게임 규칙 보존` 여부를 확인한다.
